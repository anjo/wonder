WOSVG



Element Description

A WOSVG element embeds SVG content in a HTML page.

Additional bindings will be passed onto the SVG page/component, not to the <embed> tag in the HTML page. Behaves much like WOSwitchComponent.


Synopsis

WOSVG { pageName=aPageName; directActionName=aDirectActionName; [queryDictionary=aValuesDictionary; ] filename=svgFileName; [framework=frameworkBaseName | "app" ;] width=anInt; height=anInt;... };

Bindings

pageName
Name of WebObjects SVG/XML page to display in the embed.
directActionName
The name of the direct action method (minus the "Action" suffix) to invoke to display the SVG content for the embed.
queryDictionary
NSDictionary with keys/value pairs to be placed into the URL's query string.
filename
Path to the SVG document relative to the WebServerResources directory.
framework
Framework that contains the image file. This attribute is only necessary if the image file is in a different location from the component. That is, if the component and the image file are both in the application or if the component and the image file are both in the same framework, this attribute isn't necessary. If the image file is in a framework and the component is in an application, specify the framework's name here (minus the .framework extension). If the image file should be in the application but the component is in a framework, specify the "app" keyword in place of the framework name.
width
The horizontal dimension of the SVG view.
height
The vertical dimension of the SVG view.
