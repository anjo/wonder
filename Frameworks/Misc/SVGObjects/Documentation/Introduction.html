SVGObjects Specification



Introduction

The dynamic elements and components in the SVGObjects Framework fall into 3 categories:

  • HTML components

    There is only one element in this category - WOSVG. This element is used along with other WebObjects components in HTML pages and components.

  • XML components

    These are generic XML components. They can also be used in XML documents other than SVG. (For example: they may be used to construct XHTML pages for a WebObjects application.)

  • SVG components

    These are dynamic components that generate SVG specific content.


WXGenericElement

SVG documents can be marked up in WebObjects using WOGenericContainer and WOGenericElement. But when using WOGenericElement remember to set otherTagString to "/".

For convenience, WXGenericElement has been provided. This component is virtually identical to WOGenericElement with the exception of closing its tags with a backslash. e.g: a WXGenericElement with elementName = "hr" will be rendered as <hr/> instead of <hr>.


The Style Attribute

All of the SVG elements have an optional attribute: style.
Style attributes such as fill, stroke, stroke-width, etc.. can be set as an alternative to constructing complex style strings, by prefixing the style attributes with "$" (for $tyle). e.g: 

	"$fill" = "blue";
Will produce a style string like: 
	<text style="fill:blue;"></text>
