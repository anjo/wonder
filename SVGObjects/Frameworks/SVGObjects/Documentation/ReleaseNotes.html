SVGObjects 1.0b4
Release Notes

Last Updated July 30, 2001

This document lists whats new & those problems known to exist at the time of the release.

What's new

Release 1.0b4 (for WebObjects 5)

  • Bug fixes in EZForms + ComponentTour.
  • Added support for ProjectBuilderWO for Windows 2000.

Release 1.0b3 (for WebObjects 5)

  • EZForms no longer uses a datasource and has been simplified overall.
  • The framework and examples have been re-organized to suit WebObjects 5.

Release 1.0 (for WebObjects 4.5)

  • 'pluginspage' has been added to WOSVG
  • SOPrint example has been factored out into a separate distribution 'SOPrint Framework'.

Release 1.0b1

  • The SVG* classes have been re engineered. They all subclass SODynamicElement, allowing for additional bindings to be passed onto the component. In addition any bindings of the form "$key" are appended to the style attribute.
  • The new classes have been renamed as SO* components.
  • A completed (yet not completely tested) set of SO* components have been provided for Text, Shapes, Filter, Painting & Document.
  • A new XML element has been added WXScript for inclusion of ECMA/JavaScripts into SVG.

Release 0.9.5

  • The remaining WebScript classes in SVGObjects Framework have been re-written in Java.
  • WXHyperlink & SVGImage are now feature complete WODynamicElements.
  • Minor bug fixes inWXGenericElement & WOSVG.

Release 0.9.4

  • The Hello Example has been expanded to include more typographic styles than the two in the "Getting Started with SVG and WebObjects" article.

Known Problems in This Release

SVGObjects Framework

Reference #001
Problem Vending embedded SVG content via directActions does not work.
Description When using WOSVG to embed SVG content in a page, vending the SVG content as a directAction may not work if there are form values. In particular, if the directAction URL is 'too long' the SVG will not display correctly.
Workaround     Use non embedded svg using frames or vend the svg as a WebObjects component action.
   
Reference #002
Problem DirectActions in XMLHyperlinks don't work in SVG/XML documents
Description XMLHyperlinks to directActions in SVG documents won't work.
Workaround     Use component actions or resolved URLs using the href binding.
